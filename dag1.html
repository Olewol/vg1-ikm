🍎 Mac- og maskinvare-guide

⚠️
Les dette FØRST! Ikke alle maskiner støtter VMware Workstation/Player. Finn din maskin i tabellen under.
MaskinProsessorAnbefalt løsning
Windows-PCIntel / AMD✅ VMware Workstation Player (gratis)
Mac (før 2020)Intel✅ VMware Fusion Player (gratis for personlig bruk)
Mac (2020 og nyere)Apple M1/M2/M3⚠️ Se egen seksjon under
Skole-PC (stasjonær)Intel / AMD✅ VMware allerede installert
🍎
Mac med Apple Silicon (M1/M2/M3) – tre alternativer:

Alt 1 – UTM (gratis, anbefalt): Last ned på mac.getutm.app. Kjører Ubuntu ARM64. Begrensning: Windows 10/11 (x86) støttes ikke.

Alt 2 – Parallels Desktop (~kr 900/år): Støtter Windows 11 ARM og gir best ytelse. Anbefales kun hvis skolen har lisens.

Alt 3 – Lab-PC på skolen: Stasjonære PC-er i lab har VMware installert. Du kan ikke jobbe hjemme, men kan lese teorien på nettsiden hjemme.

🛠️ Lag din egen VM med Windows 11

📁
Windows 11 ISO-fil finner du i Teams – last den ned til maskinen din før du begynner.
💡
Last ned VirtualBox gratis fra virtualbox.org – velg macOS-versjonen.

  1. Åpne VirtualBox og trykk «New»

  2. Gi maskinen et navn, f.eks. Windows11-VM

  3. Under ISO Image – klikk på mappen og velg Windows 11 ISO-filen du lastet ned fra Teams

  4. Kryss av for «Skip Unattended Installation» for å installere manuelt

  5. Sett RAM til minst 4 GB (4096 MB) og prosessorer til 2

  6. Sett virtuell disk til minst 64 GB

  7. Trykk «Finish» og deretter «Start»

  8. Følg Windows 11-installasjonsveiviseren

💡
VMware Workstation Pro er nå gratis for personlig bruk. Last ned fra vmware.com.

  1. Åpne VMware Workstation og velg «Create a New Virtual Machine»

  2. Velg «Typical (recommended)» og klikk Next

  3. Velg «Installer disc image file (iso)», klikk Browse og finn Windows 11 ISO-filen fra Teams

  4. VMware oppdager automatisk Windows 11 – fyll inn produktnøkkel (kan hoppes over) og velg versjon

  5. Gi VM-en et navn og velg lagringssted

  6. Sett disk til minst 64 GB

  7. Klikk Finish, og gå deretter til «Edit virtual machine settings» → Processors – sett Number of processors: 1 og Number of cores per processor: 4

  8. Start VM-en – Windows 11 installeres automatisk

⚙️
Anbefalt konfigurasjon for VirtualBox på Windows – disse innstillingene gir mye bedre ytelse:

  1. Opprett VM-en på samme måte som i Mac-guiden over (velg Windows 11 ISO fra Teams)

  2. Prosessorer: Sett antall CPU-kjerner til 4 (Settings → System → Processor)

  3. Skjermkort: Gå til Settings → Display og aktiver «Enable 3D Acceleration» – sett Video Memory til maks (128 MB)

  4. Virtuell disk: Gå til Settings → Storage, klikk på den virtuelle harddisken, og kryss av for «Solid-state Drive»

  5. Trykk OK, start VM-en og følg Windows 11-installasjonsveiviseren

✅
Disse tre innstillingene (4 kjerner, 3D-akselerasjon og SSD) gjør at Windows 11 kjører vesentlig raskere i VirtualBox.

💡 Tips under Windows 11-installasjonen

⌨️
Trykk en tast for å starte! Når VM-en starter opp for første gang, viser den meldingen «Press any key to boot from CD or DVD» – du har bare noen sekunder. Følg med på skjermen og trykk en vilkårlig tast raskt, ellers starter den ikke fra ISO-filen.
🔑
Produktnøkkel – velg «Har ikke produktnøkkel»
Når installasjonen spør om produktnøkkel, velg «I don't have a product key» (eller «Har ikke produktnøkkel»). Du trenger ikke nøkkel for å bruke Windows 11 i skolesammenheng.
🪟
Velg Windows 11 Pro
Når du får valg om Windows-versjon, velg Windows 11 Pro. Pro-versjonen har støtte for domenetilkobling og Active Directory som vi bruker senere.
👤
Førstegangsoppsett – lag lokal bruker
Etter installasjonen vil Windows prøve å koble deg til internett og en Microsoft-konto. Velg i stedet å lage en lokal bruker.

Hvis du sliter med å komme deg forbi internett-kravet eller ikke finner knappen for lokal konto, prøv dette:
  1. Åpne Command Prompt med Shift + F10
  2. Skriv inn kommandoen: start ms-cxh:localonly
  3. Trykk Enter – da åpnes veiviseren for lokal konto direkte

🧠 Teori – Virtualisering

📚 Les og forstå – ikke noe som skal leveres inn

En virtuell maskin (VM) er en programvarebasert datamaskin som kjører inne i en ekte, fysisk datamaskin. Den oppfører seg akkurat som en ekte maskin – med eget operativsystem, egne filer og egne nettverksinnstillinger.

💡
Huskeregel: Den fysiske maskinen er som et hus. En virtuell maskin er som en leilighet inni huset – egne rom, egne nøkler, sin egen inngang – men deler selve bygningen.
BegrepForklaring
Vert (host)Den fysiske maskinen som kjører VMware
Gjest (guest)Den virtuelle maskinen som kjører inni verten
HypervisorProgramvaren som styrer og isolerer VM-ene
SnapshotEt «fotografi» av VM-ens tilstand – du kan gå tilbake til dette punktet
ISO-filEn digital kopi av en installasjons-CD/DVD – brukes til å installere OS
VMDKFilen som fungerer som harddisken til den virtuelle maskinen

Hvorfor bruker vi virtuelle maskiner?

🧪
Testing
Prøv ut ny programvare uten å risikere den ekte maskinen.
🖥️
Serverdrift
Kjør mange servere på én fysisk maskin.
🎓
Opplæring
Eksperimenter trygt – nullstill med snapshot.
🔒
Sikkerhet
Isoler potensielt farlig programvare.

Type 1 vs. Type 2 hypervisor

🏢
Type 1 – Bare-metal
Kjører direkte på maskinvaren. Eksempler: VMware ESXi, Microsoft Hyper-V. Brukes i profesjonelle datasentre.
💻
Type 2 – Hosted
Kjører som et vanlig program på et eksisterende OS. Eksempler: VMware Workstation, VirtualBox, UTM. Dette er det vi bruker!

🎧 Lydvennlig oppsummering

🎧
En virtuell maskin er en datamaskin som lever inne i en annen datamaskin. Programvaren som gjør dette mulig kalles en hypervisor. VMware Workstation er en type 2-hypervisor som kjører som et vanlig program. Når du oppretter en VM, lager du en fil på harddisken som oppfører seg som en hel datamaskin. Du kan alltid gå tilbake til en kjent tilstand med snapshot.

💻 Oppgaver – Dag 1

📎
Disse oppgavene skal dokumenteres. Fyll inn svar og skjermbilder i dokumentasjonsmalen.
🟢
Oppgave 1.1 – Start din første VM (Basisoppgave)
Mål: Installere og starte en forhåndskonfigurert VM i VMware.

  1. Åpne VMware Workstation/Player

  2. Velg «Open a Virtual Machine» og finn filen læreren har delt (.ova-fil)

  3. Trykk «Play virtual machine»

  4. Vent til Windows starter opp – logg inn med passordet læreren har gitt

  5. Ta et skjermbilde av den kjørende VM-en

  1. Åpne UTM

  2. Velg «Create a New Virtual Machine» → «Virtualize»

  3. Velg Linux → last inn Ubuntu ARM ISO

  4. Tildel 2 GB RAM og 20 GB disk

  5. Start VM-en og følg Ubuntu-installasjonen

📸
Krav til dokumentasjon:
• Skjermbilde av VM i kjøring
• Svar: Hva er forskjellen mellom vert og gjest i ditt oppsett?
🟢
Oppgave 1.2 – Utforsk VMware-grensesnittet (Basisoppgave)
Mål: Lære de viktigste funksjonene – snapshot og hardware-innstillinger.

  1. Ta en Snapshot av VM-en mens den kjører (VM-menyen → Snapshot → Take Snapshot)

  2. Gjør en liten endring i VM-en (f.eks. endre bakgrunnsbilde)

  3. Gå tilbake til snapshot og sjekk at endringen er borte

  4. Finn ut hvor mye RAM og CPU VM-en din er tildelt (Settings → Hardware)

📸
Krav til dokumentasjon:
• Forklar med egne ord hva en snapshot er og hvorfor det er nyttig
• Skriv ned RAM og CPU-tildeling
🔵
Fordypningsoppgave 1.F – Installer Ubuntu Desktop fra ISO
For deg som har kontroll på basisoppgavene:
Last ned Ubuntu Desktop 22.04 LTS ISO fra ubuntu.com, opprett ny VM (New → Typical), velg ISO-fil, tildel minst 2 GB RAM og 20 GB disk, fullfør installasjonen, og installer VMware Tools.

🧠 Teori – Brukeradministrasjon

🔒
Principle of Least Privilege – En bruker skal bare ha akkurat den tilgangen de trenger for å gjøre jobben sin. Dette er et grunnleggende sikkerhetsprinsipp i all IT-administrasjon.
BrukertypeBeskrivelse
AdministratorFull kontroll. Kan installere programmer og endre systeminnstillinger.
StandardbrukerKan bruke programmer og endre egne innstillinger, men ikke installere software.
GjestSvært begrenset tilgang. Vanligvis deaktivert av sikkerhetsgrunner.
PowerShell / CMD
net user                          # Vis alle brukere
net user <navn> <passord> /add    # Opprett bruker
net user <navn> /delete           # Slett bruker
net localgroup Administrators <bruker> /add  # Admin-gruppen
Bash
sudo adduser <brukernavn>          # Opprett bruker
sudo deluser --remove-home <navn>  # Slett med hjemmappe
sudo passwd <brukernavn>          # Endre passord
sudo usermod -aG sudo <bruker>   # Gi sudo-rettigheter
cat /etc/passwd                    # Vis alle brukere
su - <brukernavn>                  # Bytt til bruker
💡
sudo lar deg utføre enkeltkommandoer med administratorrettigheter uten å være logget inn som root. Tryggere enn å jobbe som root.

💻 Oppgaver – Dag 3

🟢
Oppgave 3.1 – Lokale brukere i Windows (GUI)
Åpne Datamaskinbehandling → Lokale brukere og grupper → Brukere. Opprett: elev_anna (standard), elev_bjorn (standard), it_admin (legg til i Administrators). Passord: Passord123!. Logg inn som elev_anna og observer begrenset tilgang.
🟢
Oppgave 3.2 – PowerShell-brukeradministrasjon
PowerShell (Kjør som administrator)
net user                                    # 1. Vis alle brukere
net user testbruker Passord123! /add        # 2. Opprett bruker
net user testbruker                         # 3. Se detaljer
net localgroup Users testbruker /add        # 4. Legg i Users-gruppe
net user testbruker /delete                 # 5. Slett brukeren
net user                                    # 6. Bekreft sletting
🟢
Oppgave 3.3 – Brukere i Ubuntu
Bash
sudo adduser praksis_elev                  # 1. Opprett bruker
cat /etc/passwd | grep praksis_elev        # 2. Verifiser
sudo usermod -aG sudo praksis_elev        # 3. Gi sudo-rettigheter
groups praksis_elev                        # 4. Sjekk gruppemedlemskap
su - praksis_elev                          # 5. Bytt til bruker
whoami                                     # 6. Hvem er jeg?
sudo ls /root                             # 7. Test sudo
exit                                       # 8. Gå tilbake
sudo deluser --remove-home praksis_elev   # 9. Slett

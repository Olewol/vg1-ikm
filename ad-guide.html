VG1 IKM

🖥️ Active Directory – Windows Server 2025

Brukeradmin på DC01

Opprett OUer, brukere og grupper i Active Directory. Sett NTFS-rettigheter via GUI og PowerShell. Alt skjer isolert i din egen VM.

🔒 DC01 – skolelab.local
📡 192.168.10.10 – LAN Segment
🔐 Ingen internett – isolert fra skolenettet
🔒
Alt du gjør i denne guiden skjer inne i VMware på din lab-PC. DC01-serveren er ikke koblet til internett og påvirker ingenting utenfor din egen VM.

Del 1 – Logg inn og åpne ADUC

  1. Åpne VMware Workstation og start VM-en kalt DC01

  2. Logg inn: Brukernavn Administrator (eller SKOLELAB\Administrator), Passord: Skole2025!

  3. Åpne ADUC via Server Manager → Tools → Active Directory Users and Computers eller Win+R → dsa.msc

Del 2 – Opprett OU-struktur

Vi bygger denne strukturen:

Struktur
skolelab.local
├── Elever
│   ├── VG1
│   └── VG2
├── Ansatte
│   ├── Larere
│   └── IT-admins
└── Grupper

  1. Høyreklikk skolelab.local → New → Organizational Unit

  2. Navn: Elever – la «Protect from accidental deletion» være huket av ✅ – klikk OK

  3. Gjenta for Ansatte og Grupper

  4. Høyreklikk Elever → New → OU → VG1, gjenta for VG2

  5. Høyreklikk Ansatte → New → OU → Larere og IT-admins

PowerShell (Administrator)
# Hoved-OUer
New-ADOrganizationalUnit -Name "Elever" -Path "DC=skolelab,DC=local"
New-ADOrganizationalUnit -Name "Ansatte" -Path "DC=skolelab,DC=local"
New-ADOrganizationalUnit -Name "Grupper" -Path "DC=skolelab,DC=local"

# Under-OUer
New-ADOrganizationalUnit -Name "VG1" -Path "OU=Elever,DC=skolelab,DC=local"
New-ADOrganizationalUnit -Name "VG2" -Path "OU=Elever,DC=skolelab,DC=local"
New-ADOrganizationalUnit -Name "Larere" -Path "OU=Ansatte,DC=skolelab,DC=local"
New-ADOrganizationalUnit -Name "IT-admins" -Path "OU=Ansatte,DC=skolelab,DC=local"

# Verifiser
Get-ADOrganizationalUnit -Filter * | Select-Object Name, DistinguishedName

Del 4 – Grupper og medlemmer

PowerShell
# Opprett grupper
New-ADGroup -Name "GRP-Elever-VG1" -GroupScope Global -GroupCategory Security `
  -Path "OU=Grupper,DC=skolelab,DC=local"

New-ADGroup -Name "GRP-Larere" -GroupScope Global -GroupCategory Security `
  -Path "OU=Grupper,DC=skolelab,DC=local"

New-ADGroup -Name "GRP-IT-admins" -GroupScope Global -GroupCategory Security `
  -Path "OU=Grupper,DC=skolelab,DC=local"

# Legg til brukere
Add-ADGroupMember -Identity "GRP-Elever-VG1" -Members "anna.olsen","bjorn.hansen"
Add-ADGroupMember -Identity "GRP-Larere" -Members "kari.nygaard"
Add-ADGroupMember -Identity "GRP-IT-admins" -Members "it.admin"

# Verifiser
Get-ADGroupMember -Identity "GRP-Elever-VG1" | Select-Object Name, SamAccountName

Del 5 – NTFS-rettigheter på delt mappe

💡
GRP-Elever-VG1 får kun lese på C:\Delt\Elever. GRP-Larere får full tilgang på C:\Delt\Larere. Dette er least privilege i praksis.

  1. Åpne File Explorer → C:\ → Opprett mappe Delt → inni den: Felles, Larere, Elever

  2. Høyreklikk Elever → Properties → fanen Security → Edit → Add

  3. Skriv GRP-Elever-VG1 → Check Names → OK

  4. Sett: Read & execute ✅ List folder ✅ Read ✅ | Write ❌ Modify ❌ Full control ❌

  5. Gjenta for Larere-mappen, legg til GRP-Larere med Full control ✅

PowerShell
# Opprett mappestruktur
New-Item -ItemType Directory -Path "C:\Delt\Felles" -Force
New-Item -ItemType Directory -Path "C:\Delt\Larere" -Force
New-Item -ItemType Directory -Path "C:\Delt\Elever" -Force

# Lesetilgang for elever
$acl = Get-Acl "C:\Delt\Elever"
$regel = New-Object System.Security.AccessControl.FileSystemAccessRule(
  "SKOLELAB\GRP-Elever-VG1", "ReadAndExecute",
  "ContainerInherit,ObjectInherit", "None", "Allow")
$acl.SetAccessRule($regel)
Set-Acl -Path "C:\Delt\Elever" -AclObject $acl

# Full tilgang for lærere
$acl2 = Get-Acl "C:\Delt\Larere"
$regel2 = New-Object System.Security.AccessControl.FileSystemAccessRule(
  "SKOLELAB\GRP-Larere", "FullControl",
  "ContainerInherit,ObjectInherit", "None", "Allow")
$acl2.SetAccessRule($regel2)
Set-Acl -Path "C:\Delt\Larere" -AclObject $acl2

# Verifiser
(Get-Acl "C:\Delt\Elever").Access | Select-Object IdentityReference, FileSystemRights

Del 6 – Nyttige administrasjonskommandoer

PowerShell – Brukeradministrasjon
# Tilbakestill passord
Set-ADAccountPassword -Identity "anna.olsen" -Reset `
  -NewPassword (ConvertTo-SecureString "NyttPassord2025!" -AsPlainText -Force)

# Deaktiver / aktiver konto
Disable-ADAccount -Identity "bjorn.hansen"
Enable-ADAccount -Identity "bjorn.hansen"

# Finn alle deaktiverte kontoer
Search-ADAccount -AccountDisabled | Select-Object Name, SamAccountName

# Se alle brukere i en OU
Get-ADUser -Filter * -SearchBase "OU=VG1,OU=Elever,DC=skolelab,DC=local" | Select-Object Name, Enabled

# Flytt bruker til annen OU
Move-ADObject -Identity "CN=Anna Olsen,OU=VG1,OU=Elever,DC=skolelab,DC=local" `
  -TargetPath "OU=VG2,OU=Elever,DC=skolelab,DC=local"
